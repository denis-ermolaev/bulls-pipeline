## Настройка окружения
Для автоматической подготовки окружения используется команда:

```bash
# Для тестового окружения
make test_project_preparation

# Скачивает все данные (Около 30гб)
make project_preparation
```
- Устанавливает uv (менеджер пакетов), если ещё не установлен
- Устанавливает Python нужной версии, если ещё нет
- Синхронизирует зависимости Python через uv, настраивает pre-commit-хуки
- Скачивает публичные данные(реф. панель, реф. геномы, карты рекомбинации, бинарные программы для работы скрипта(plink, magma и т.п))
- Проверяет наличие podman. **Его нужно установить самостоятельно (`sudo apt install -y podman`)** (используется для контейниризации - фиксации окружения)
- Собирает образ контейнера с нуля или из загруженного архива.

## Настройки / Управление параметрами

Все пути и параметры задаются в файле `.env`. Благодаря контейнеризации (в разработке) монтирование томов и проброс переменных будут выполняться автоматически.

`config.yaml` - конфиг для управления пайплайном

## Использование
Запуск команд через Makefile, можно с помощью GUI - расширения: "vscode-makefile-term", "andrevops.makestro" для VScode.

`make status` - проверить запущен ли контейнер

- `make build` - после изменения Dockerfile
- `make run_container` - запуск podman контейнера
- `make run_pipeline` - **запустить пайплайн**
- `make stop` - в конце работы, чтобы остановить контейнер

Посмотреть потребление ресурсов - `make stats`

## Разработка

1. Внутри контейнера

`make run_container_with_ssh` - запускает контейнер с доступом по ssh, полученный порт можно перебросить на локальную машину и с VScode зайти по ssh (`ssh root@localhost -p PORT`).


2. Снаружи, а запуск скриптов и jupyter внутри
- `make install_host` - установка python зависимостей на хост, для работы языкового сервера в IDE
- Работа с Jupyter внутри контейнера `make run_jupyter` -> адрес в консоле -> в vs code -> выбрать другое ядро -> существующий jupyter сервер -> вписать адрес.
- `make run_pipeline` запуск пайплайна или отдельных частей

3. Общее:
- Запуск тестов `make test`
- Периодически `make clean` - для удаления старых образов после `make build`
